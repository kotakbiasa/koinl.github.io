Bab 2 Operasi I/O Disk

Kategori: Hadoop
Label:

Tanggal: 07-09-2022 12:20:56

Dalam suatu sistem, ketika data yang perlu diolah jumlahnya sangat besar, kemungkinan terjadinya kerusakan data jauh lebih besar. Oleh karena itu perlu dilakukanPemeriksaan integritas data；
Hadoop mengadopsi RPC Untuk mencapai komunikasi antar-proses, itu Mekanisme serialisasi yang dapat ditulis. Serialisasi memiliki dua tujuan:Komunikasi antar proses dan penyimpanan persistensi data。
File dapat dikompresi sebelum dikirim. Hal ini memiliki dua manfaat utama, yang juga penting untuk memproses data dalam jumlah besar:Mengurangi ruang penyimpanan dan mempercepat transfer data pada jaringan dan disk。
Untuk beberapa aplikasi, struktur data khusus sering kali diperlukan untuk menyimpan data, sehingga Hadoop telah mengembangkan wadah tingkat yang lebih tinggi untuk tujuan ini: SequenceFile dan Mapfile.

1. Integritas data

Secara umum, Anda dapat menggunakanVerifikasi checksumcara untuk memeriksa integritas data.

Checksum mengacu pada jumlah yang digunakan untuk memverifikasi sekumpulan item data terminal di bidang pemrosesan data dan komunikasi data.
Checksum akan dibuat sebelum data dikirim. Saat data dikirimkan, checksum akan dihitung kembali. Jika tidak konsisten maka akan ditentukan datanya.rusak。
Selain itu, checksum juga dihitung ketika sistem file di-restart.

1. Integritas data HDFS

Selain menghitung checksum verifikasi saat menulis dan membaca untuk memverifikasi integritas data, HDFS juga secara berkala menghitung checksum blok untuk memverifikasi integritas data.

1) Verifikasi saat menulis data ke HDFS

Klien mengirimkan permintaan tulis ke HDFS, dan DataNode terakhir yang menerima salinan blok akan menghitung checksum dan membandingkannya dengan checksum yang dikirim oleh Klien. Jika informasi yang sama disimpan dan dikembalikan, kesalahan IOException akan diberikan ke Klien.
Selain itu, ketika DataNode menyalin data dari node data lain, DataNode juga akan memverifikasi data dengan membandingkan checksum.

2) Verifikasi saat membaca data dari HDFS

Setiap node data akan mempertahankan waktu checksum dan pembaruan checksum setelah setiap verifikasi ke log.
Ketika Klien membaca data dari node data, Klien menempatkan data tersebut di cache Klien, kemudian menghitung ulang checksum dan membandingkannya dengan checksum terbaru yang disimpan di node data.

3) Deteksi reguler proses daemon latar belakang DataNode

Ini memecahkan masalah kerusakan media penyimpanan fisik. Setiap node data akan menjalankan proses DataBlockScanner di latar belakang. Proses ini secara berkala memeriksa blok-blok pada node data sehingga blok-blok yang rusak dapat dideteksi dan diperbaiki tepat waktu sebelum Klien membacanya.
Selain itu, ketika checksum replika blok ditemukan tidak konsisten dengan data di log, node data akan menandainya sebagai rusak. Ketika node data mengirimkan detak jantung ke node metadata, node metadata membiarkan node data menangani blok data yang rusak.

2. Serialisasi dan Deserialisasi

Serialisasi Java adalah proses mengubah objek Java menjadi rangkaian byte, dan deserialisasi Java adalah proses memulihkan rangkaian byte ke objek Java.
Misalnya, membuat serial objek gambar Java, mengirimkannya melalui jaringan, dan terakhir melakukan deserialisasi untuk mendapatkan objek yang diperlukan.

1. Serialisasi

Sederhananya, serialisasi adalah mengubah objek ke dalam format yang nyaman untuk transmisi, seperti biner, array byte, json, xml, dll. Hadoop menggunakan Writable.

3. Kompresi data

Jumlah data pada Hadoop terlalu besar sehingga perlu dilakukan kompresi file untuk mengurangi ruang penyimpanan yang digunakan untuk penyimpanan file serta mempercepat transmisi data pada jaringan dan disk.

1. Metode kompresi dan dekompresi Codec

1) Antarmuka CompressionCodec

Antarmuka CompressionCodec mendefinisikan metode kompresi dan dekompresi data, yaitu createOutputStream() dan createInputStream().
Dengan menggunakan metode createOutputStream(), Anda dapat membuat objek CompressionOutputStream baru untuk data yang tidak terkompresi dan mengompresi data dalam aliran output.
Dengan menggunakan metode createInputStream(), data yang dibaca dari aliran input dapat didekompresi, dan objek CompressionInputStream baru dibuat dari data yang didekompresi.

Menjalankan metode pengujian Compressor() di Eclipse dapat menghasilkan file terkompresi (seperti "c.txt.gz") di lokasi tertentu pada sistem lokal (seperti "E:/data");
Jalankan metode pengujian UnCompressor(), dan konten file yang diperoleh dengan mendekompresi file terkompresi akan ditampilkan di jendela "Konsol" di bawah.

2) CompressionCodecFactory

Saat membaca file terkompresi, kelas CompressionCodecFactory dapat menyimpulkan algoritma Codec mana yang akan digunakan berdasarkan ekstensi file.

Dengan menggunakan metode getCodec() yang disediakan oleh CompressionCodecFactory, Anda bisa menerima objek Path dan memetakan ekstensi file ke metode Codec yang sesuai.
Jalankan metode pengujian FileDecompressor() di Eclipse untuk mendapatkan file yang didekompresi di direktori yang sama dengan file terkompresi (yaitu hdfs://hadoop00:9000/mywork/test/c.txt.gz).

2. Kompresi dan fragmentasi masukan

Jumlah data di Hadoop terlalu besar, dan diperlukan dukungan segmentasi setelah kompresi. Misalnya, jika file berukuran 2 GB dibagi menjadi 16 blok (setiap blok berukuran 128 MB secara default), jika file tersebut digunakan sebagai input tugas MapReduce, 16 shard akan dibuat, dan setiap shard akan memulai tugas Map untuk memproses data shard secara mandiri.

Dengan asumsi ukuran file terkompresi adalah 1,6 GB, jumlah blok akan dikurangi menjadi 13, dan jumlah tugas Peta juga akan dikurangi, sehingga meningkatkan efisiensi kerja.
Tentu saja format kompresinya harus mendukung segmentasi (seperti bzip2). Bagi yang tidak mendukung segmentasi, emmm.

四、Hadoop 文件的数据结构

Diantaranya, SequenceFile dan MapFile lebih umum. File jenis MapFile diurutkan file SequenceFile dengan indeks.

1. File Urutan

adalah file biner serial, diakhiri denganMetode pasangan nilai kunciCatat datanya.

Jalankan metode pengujian SequemceFileWriteDemo() di Eclipse untuk menulis data ke file SequemceFile (yaitu hdfs://hadoop00:9000/mywork/test/sequencedemo.seq).

Di mesin virtual, Anda dapat melihat konten file dengan menjalankan perintah berikut:hdfs dfs -text /mywork/test/sequencedemo.seq

Jalankan metode pengujian SequemceFileReadDemo() di Eclipse untuk membaca data dari file SequemceFile, dan log akan ditampilkan di jendela "Konsol" di bawah.

2. File Peta

Jalankan metode pengujian MapFileWriteDemo() di Eclipse untuk menulis data ke file MapFile (yaitu hdfs://hadoop00:9000/mywork/test/mapfiledemo.map).

Di mesin virtual, Anda dapat melihat indeks dan isi data file masing-masing dengan menjalankan perintah berikut:hdfs dfs -text /mywork/test/mapfiledemo.map/index Dan hdfs dfs -text /mywork/test/mapfiledemo.map/data。

Jalankan metode pengujian MapFileReadDemo() di Eclipse untuk membaca data dari file MapFile. Saat ini, seluruh konten file dan nilai kunci 10 akan ditampilkan di jendela "Konsol" di bawah.
