Kategori database dibagi menjadi database sistem dan database pengguna.
Basis data sistem dibagi menjadi master, msdb, tempdb, model, dan sumber daya, yang masing-masing menyimpan informasi sistem, informasi pekerjaan, informasi sementara, templat, hanya-baca, dan tidak dapat dilihat.
Simpan database sebagai file. Dibagi menjadi file data dan file log.
File data dibagi menjadi file data primer (hanya satu mdf, ≥3) dan file data sekunder (ndf, n)
berkas log (ldf, ≥1)
Alokasi ruang penyimpanan database:
Grup file: grup file utama dan grup file pengguna

1. Membuat dan memelihara database

1. SQL Server lima database

master:记录所有系统信息，包括实例范围的元数据、端点、连接服务器和系统配置设置。还记录了所有其他数据库的存在、数据库文件的位置以及SQLServer的初始化信息
model：用作实例上创建的所有数据库的模板。当用户创建一个数据库时，系统将自动将Model数据库中的全部内容复制到新建数据库中
temodb:临时数据库，用于保存临时对象或中间结果集，并未数据的排序等操作提供一个临时工作空间
msdb:供SQL Server 代理服务调度博阿金和作业以及记录操作员时使用，保存关于调度报警、作业、操作员等信息。作业是在SQLserver中定义的自动执行的一系列操作的几何，作业的执行不需要人工干预
Resource：是一个只读数据库，包含了所有系统对象，SQLServer系统对象在物理上保存在Resource数据库中，但在逻辑上却显示在每个数据库的sys架构中

2. Arsitektur

1 konsep

Ini adalah namespace logis di bawah database, yang dapat menyimpan objek database seperti tabel dan tampilan. Ini adalah wadah untuk objek database.

2 Tentukan arsitekturnya

Buat Skema Schema_name Otorisasi nama_pengguna

3 Hapus skema

Hapus Skema Schema_name Cascade|Batasi
Cascade: Saat menghapus skema, semua objek skema dalam skema juga akan dihapus.
Batasi: Menolak penghapusan jika skema yang dihapus berisi objek skema

3. Tabel partisi

1 konsep

Ini membagi data dalam tabel menjadi himpunan bagian yang berbeda secara horizontal, dan himpunan bagian data ini disimpan dalam satu atau lebih grup file dalam database. Gunakan partisi untuk mengelola dan mengakses subkumpulan data dengan cepat dan efisien, membuat tabel atau indeks besar lebih mudah dikelola

2 Buat tabel partisi

(1) Kondisi

Jika ada sejumlah besar data dalam tabel yang sering digunakan, dan operasinya pada dasarnya sama, sebaiknya tidak menggunakan tabel yang dipartisi.
Jika jumlah datanya besar, datanya tersegmentasi, dan operasi yang berbeda digunakan pada segmen data yang berbeda, maka cocok untuk menggunakan tabel yang dipartisi.
(2) Langkah

5. Tampilan indeks

1 Konsep dasar

Tampilan standar juga merupakan tabel virtual, karena format kumpulan hasil yang dikembalikan oleh tampilan ini pada dasarnya sama dengan tabel dasar, terdiri dari kolom dan baris, dan cara mengindeks tampilan dalam pernyataan SQL sama dengan cara mengindeks tabel dasar. Kumpulan hasil tampilan standar tidak disimpan secara permanen dalam database. Setiap kali data ditanyakan melalui tampilan standar, sistem manajemen database akan secara internal mengganti definisi tampilan dengan pernyataan kueri untuk tabel dasar, dan kemudian mengkueri tabel dasar.

2 Kesempatan yang cocok untuk menetapkan tampilan yang diindeks

如果很少更新基础数据，则索引视图的效果最佳
如果经常更新基础数据，则维护索引视图的成本可能超过使用索引视图带来的性能收益
如果基础数据以批处理的形式定期更新，但是在更新之间主要作为只读数据进行处理，可考虑在更新前删除所有索引视图，然后再重建索引视图，这样做可以提高更新的性能
处理大量行的连接和聚合操作
许多查询经常执行的链接和聚合操作

3 Tentukan tampilan indeks

基本要求
    定义索引视图时，视图不能引用任何其他视图，只能引用索引表
    视图引用的所有基本表必须与视图位于位于同一个数据库中，并且所有者也与视图相同
    必须使用SchemaBinding选项创建视图
    视图中的表达式引用的所有函数必须是确定的
    对视图创建的第一个索引必须是唯一聚集索引，之后再创建其他的非聚集索引
