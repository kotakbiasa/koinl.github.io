Bab 6 Dasar-Dasar Jaringan

Kategori: Linux
Label:

Tanggal: 02-06-2022 10:17:29

<Catatan: Perangkat kartu jaringan semuanya merupakan contoh perangkat ens33>
Mengenai konfigurasi jaringan Linux, dapat dibagi menjadi konfigurasi IP jaringan sementara [ifconfig, ip] dan permanen [nmcli, file konfigurasi tautan jaringan].
Saya tidak akan mengesampingkan artikel ini untuk saat ini, tetapi hanya berbicara tentang keabadian.
File konfigurasi jaringan disediakan untuk tujuan informasi saja dan fokusnya adalah pada pemrosesan nmcli.

File konfigurasi jaringan: vim /etc/sysconfig/network-scripts/ifcfg-nama perangkat kartu jaringan (ens33, dll.)

Perangkat kartu jaringan itu sendiri

Konfigurasi koneksi kartu jaringan
nmcli dibagi menjadi pengaturan perangkat dan koneksi. Beberapa sambungan dapat diterapkan ke perangkat yang sama, namun hanya satu sambungan yang dapat diaktifkan secara bersamaan.

Pengaturan untuk perangkat

Periksa status perangkat

Saat menanyakan perangkat untuk kesederhanaan, gunakan nmcli device status Itu sudah cukup.
Untuk menanyakan informasi spesifik untuk semua perangkat, gunakan nmcli device show. Tentu saja, jika Anda menambahkan nama tertentu setelahnya, Anda hanya akan menanyakan informasi perangkat tersebut.
Selain itu, Anda juga dapat memutuskan atau menyambungkan perangkat.

Pengaturan untuk koneksi

Status koneksi kueri

Untuk menanyakan semua informasi secara singkat, gunakan nmcli conncetion show, menanyakan informasi detail tertentu dan menambahkan namanya. Tentu saja, ini juga dapat dinonaktifkan atau diaktifkan.

penggunaan menggambarkan
status perangkat nmcli Status peralatan
perangkat nmcli tampilkan ens33 Tampilkan status ens33
nmcli d putuskan / sambungkan ens33 perangkat ens33 menonaktifkan/mengaktifkan
acara koneksi nmcli menunjukkan koneksi
koneksi nmcli tampilkan ens33 Tampilkan koneksi jaringan ens33
koneksi nmcli turun / naik ens33 Nonaktifkan/aktifkan konfigurasi koneksi

Modifikasi konfigurasi koneksi

Untuk modifikasi tidak lebih dari penambahan, penghapusan dan modifikasi. Contoh berikut digunakan untuk menggambarkan:

Contoh 1:- Buat konfigurasi koneksi bernama conn2, tentukan alamat ip statis, tidak ada koneksi otomatis. (tugas, jenis, alamat, gerbang)
nmcli c a ifname ens33 con-name conn2
ipv4.method manual type Ethernet ipv4.addresses 192.168.85.136/24 gw4 192.168.85.2
autoconnect no

  • Setelah dicek, kita dapat melihat bahwa koneksi conn2 sudah ada, tetapi tidak terkait dengan suatu perangkat.
    nmcli c s

Contoh 2: Tunggu, lihat tabel di bawah

parameter inti Parameter file konfigurasi menggambarkan
ipv4.metode manval / otomatis BOOTPROTO=tidak ada atau statis / dhcp IP yang dialokasikan secara dinamis
ifname ens33 DEVICE=ens33 Tentukan nama kartu jaringan
ipv4.alamat 192.168.100.100/24 IPADDR=192.168.100.100'/n'PREFIX=24 Alamat IP dan bit mask
gw4 192.168.100.1(2?) GATEWAY=192.168.100.1 Siapkan gerbang
ipv4.dns 8.8.8.8 DNS0=8.8.8.8 Siapkan server DNS
ipv4.dns-pencarian example.com DOMAIN=example.com nama domain
koneksi.koneksi otomatis ya ONBOOT=yes Terhubung secara otomatis saat boot

  • Lihat file konfigurasi conn2
    cat /etc/sysconfig/network-scripts/ifcfg-conn2

  • Pertama-tama nonaktifkan koneksi ens33, lalu aktifkan koneksi conn2. Ini adalah cara berpindah jaringan jika ada dua jaringan.
    nmcli cd ens33
    nmcli c kamu koneksi2

  • Anda dapat mempelajari lebih lanjut dengan memeriksa status koneksi.
    nmcli c s

  • Lihat status perangkat ens33.
    nmcli d s
    nmcli d tampilkan ens33

  • Ubah konfigurasi koneksi conn2, atur IP ke , DNS ke , dan atur koneksi agar diaktifkan secara otomatis saat boot.
    nmcli c m conn2 ipv4.addresses 192.168.85.59/24 ipv4.dns 192.168.85.1 koneksi.koneksi otomatis ya

  • Lihat isi file konfigurasi ifcfg-conn2
    cat -n /etc/sysconfig/network-scripts/ifcfg-conn2

  • Mulai ulang layanan jaringan agar konfigurasi baru diterapkan.
    systemctl memulai ulang jaringan

  • Kembalikan jaringan ke keadaan semula
    nmcli c d koneksi2

Modifikasi file konfigurasi sebenarnya berarti memodifikasi konfigurasi koneksi.
