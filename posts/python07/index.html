Bab 7 Pemformatan File dan Data

7.1 Penggunaan file
File adalah rangkaian data yang disimpan di penyimpanan sekunder dan dapat berisi konten data apa pun. Secara konseptual, file adalah kumpulan dan abstraksi data. File termasuk file teks dan file biner.
7.1.1 Jenis file
File teks umumnya terdiri dari karakter dalam satu pengkodean tertentu, seperti pengkodean UTF-8, dan kontennya mudah untuk ditampilkan dan dibaca secara seragam. Sebagian besar file teks dapat dibuat, dimodifikasi, dan dibaca menggunakan perangkat lunak pengedit teks atau pengolah kata. karena file teks
Ada pengkodeannya, sehingga bisa juga dilihat sebagai string panjang yang disimpan di disk, seperti file teks dalam format txt
File biner secara langsung terdiri dari bit О dan bit 1. Tidak ada pengkodean karakter terpadu. Format organisasi data internal file tersebut adalah
Formatnya tergantung pada tujuan file. Biner adalah file yang informasinya dalam format non-karakter tetapi memiliki format tertentu, seperti file gambar dalam format png dan file video dalam format avi. Perbedaan utama antara file biner dan file teks adalah apakah terdapat pengkodean terpadu. File biner tidak memiliki pengkodean karakter terpadu dan hanya dapat dianggap sebagai aliran byte, bukan string.
Terlepas dari apakah file tersebut dibuat sebagai file teks atau file biner, file tersebut dapat dibuka dalam "mode file teks" dan "mode file biner", tetapi operasi setelah pembukaannya berbeda.
File teks a.txt, yang isinya "Ujian Peringkat Komputer Nasional", dibuka dalam mode teks. Di sini Anda perlu menempatkan file teks dan file program Python di direktori yang sama.
File dibaca dalam mode teks, file dikodekan untuk membentuk string, dan karakter yang bermakna dicetak; file dibuka dalam mode biner, dan file diurai menjadi aliran byte. Karena pengkodean, satu karakter dalam sebuah string diwakili oleh beberapa byte.
7.1.2 Membuka dan menutup file
Python mengadopsi prosedur operasi terpadu untuk file teks dan file biner, yaitu "buka-operasi-tutup". File di sistem operasi berada dalam status penyimpanan secara default. Anda harus membukanya terlebih dahulu. Program saat ini harus memiliki otoritas untuk mengoperasikan file tersebut. Membuka sistem file yang tidak ada dapat membuat file tersebut. File yang dibuka sudah terisi. Saat ini, proses lain tidak dapat mengoperasikan file tersebut. Isi file dapat dibaca atau ditulis ke file melalui serangkaian metode. Setelah operasi, file harus ditutup. Penutupan akan melepaskan kendali atas file dan menggunakan file tersebut untuk memulihkannya ke kondisi penyimpanannya. Pada saat ini, proses lain dapat mengoperasikan file tersebut.
Python membuka file melalui fungsi open() dan mengembalikan variabel yang beroperasi pada file tersebut. Struktur tata bahasa:
Nama variabel = buka (jalur file dan nama file, mode terbuka)
Fungsi open() memiliki dua parameter: nama file dan mode terbuka. Nama file dapat berupa nama file sebenarnya atau nama yang menyertakan path lengkap. Mode terbuka digunakan untuk mengontrol cara membuka file. Fungsi open() menyediakan 7 mode terbuka dasar.
Mode pembukaan file
r Mode baca-saja, jika file tidak ada, mengembalikan pengecualian FileNotFoundError, nilai default
w mode timpa, jika file tidak ada, buatlah, jika ada, itu akan menimpa file asli sepenuhnya.
x membuat mode tulis. Jika file tidak ada maka akan dibuat. Jika ada, pengecualian FileExistsError akan dikembalikan.
a Tambahkan mode penulisan. Jika file tidak ada maka akan dibuat. Jika ada, kontennya akan ditambahkan ke akhir file asli.
b mode file biner
t mode file teks, nilai default

  • Digunakan bersama dengan r/w/x/a, menambahkan fungsi membaca dan menulis secara simultan berdasarkan fungsi aslinya
    Mode terbuka diwakili oleh sebuah string. Menurut definisi string, tanda kutip tunggal atau tanda kutip ganda dapat digunakan. Dalam mode terbuka di atas, r, w, x, a dapat digunakan dalam kombinasi dengan b, t, + untuk membentuk cara mengekspresikan membaca dan menulis serta mode file.

7.1.3 Membaca dan menulis file
Tergantung pada metode pembukaannya, pembacaan dan penulisan file juga akan berbeda tergantung pada file teks atau metode pembukaan biner.
f.read(size=-1): Membaca seluruh konten file dari file. Argumennya opsional. Jika diberikan, aliran string atau byte dengan panjang hingga ukuran akan dibaca.
f.readline(size=-1): Membaca sebaris konten dari file. Argumennya opsional. Jika diberikan, aliran string atau byte berukuran panjang sebelum baris dibaca.
f.readlines(hint=-1): Baca semua baris dari file dan bentuk daftar dengan elemen di setiap baris. Argumennya opsional. Jika diberikan, baris petunjuk dibacakan.
f.seek(offset): Mengubah posisi penunjuk operasi file saat ini. Nilai offset: o adalah awal file; 2 adalah akhir file.
Jika file dibuka sebagai file teks, sebuah string dibaca; jika file dibuka sebagai file biner, aliran byte dibaca.
Jika file tidak besar, Anda dapat membaca isi file sekaligus dan menyimpannya ke variabel internal program. f.read() adalah fungsi yang paling umum digunakan untuk membaca file sekaligus. Hasilnya adalah sebuah string.
f.write(s): Tulis aliran string atau byte ke file
f.writelines(lines): Tulis daftar yang elemennya merupakan string ke file secara keseluruhan
f.write(s) menulis string s ke file. Setelah setiap penulisan, penunjuk tulis akan direkam. Metode ini dapat dipanggil berulang kali, menulis konten secara batch setelah menulis pointer, hingga file ditutup.
Saat menggunakan f.write(s), Anda harus secara eksplisit menggunakan "\n" untuk membagi teks tertulis menjadi beberapa baris. Jika Anda tidak memutus garis, string yang ditulis setiap kali akan digabungkan.
f.writelines(lines) secara langsung menghubungkan elemen tipe daftar dan menulisnya ke file f.
7.2 Dimensi organisasi data
Sekumpulan data perlu diorganisasikan sampai batas tertentu sebelum diproses oleh komputer untuk menunjukkan hubungan dasar dan logika antar data, sehingga membentuk “dimensi data”. Menurut perbedaan hubungan antar data, organisasi data dapat dibagi menjadi: data satu dimensi, data dua dimensi, dan data dimensi tinggi.
7.2.1 Data satu dimensi
Data satu dimensi terdiri dari data terurut atau tidak terurut dalam hubungan peer-to-peer dan disusun secara linier, sesuai dengan konsep array dalam matematika. Misalnya, daftar kota di Tiongkok dapat direpresentasikan sebagai data satu dimensi, dan data satu dimensi memiliki karakteristik linier.
Data dimensi sangat umum; segala sesuatu yang muncul sebagai suatu barisan atau himpunan dapat dianggap sebagai data satu dimensi.
7.2.2 Data dua dimensi
Data dua dimensi, juga disebut data tabular, terdiri dari data hubungan terkait dan disusun dalam tabel dua dimensi, sesuai dengan matriks dalam matematika. Tabel umum adalah data dua dimensi.
Misalnya, data harga konsumen yang dikeluarkan oleh Badan Statistik Nasional merupakan data dua dimensi. Arah horizontal tabel mencakup "indikator", "2014", "2015" dan "2016", yang masing-masing mewakili dimensi proyek dan waktu. Arah vertikal tabel mencakup titik-titik indikator tertentu, membentuk hubungan data dua dimensi. Diantaranya, baris pertama merupakan bagian deskripsi, yang dapat dianggap sebagai deretan data dua dimensi yang konsisten dengan baris lainnya, atau dapat juga dianggap sebagai bagian numerik selain data.
7.2.3 Data berdimensi tinggi
Data berdimensi tinggi terdiri dari data pasangan nilai kunci, disusun secara objek, dan dapat disarangkan secara hierarki.
Data berdimensi tinggi sangat umum digunakan dalam sistem Web dan merupakan cara utama untuk mengatur konten di Internet saat ini. Data berdimensi tinggi memperoleh struktur tata bahasa dari organisasi data tertentu seperti HTML, XML, dan JSON.
Mengambil format JSON sebagai contoh, berikut ini adalah representasi data berdimensi tinggi yang mendeskripsikan "buku ini", dengan tanda titik dua (:) membentuk pasangan nilai kunci, dan koma (,) memisahkan pasangan nilai kunci. Dalam format JSON, [] mengatur setiap pasangan nilai kunci menjadi satu kesatuan, membentuk pasangan nilai kunci tingkat tinggi dengan "buku ini". Data berdimensi tinggi dapat mengekspresikan sistem produksi data yang lebih fleksibel dan kompleks dibandingkan data satu dimensi dan data dua dimensi.
7.3.1 Representasi data satu dimensi
Data satu dimensi adalah jenis organisasi data yang paling sederhana. Ini adalah struktur linier dan terutama diekspresikan dalam bentuk daftar dalam bahasa Python. Misalnya, data kotamadya Tiongkok dapat diwakili oleh variabel daftar.

7.3.3 Pengolahan data satu dimensi
Untuk memproses data satu dimensi, pertama-tama Anda perlu membaca data satu dimensi dari file berformat CSV dan merepresentasikannya sebagai objek daftar. Perlu diperhatikan bahwa saat memperoleh konten dari file CSV, karakter baris baru ("'n") disertakan setelah elemen terakhir. Untuk ekspresi dan penggunaan data, baris baru ini mubazir. Anda perlu menggunakan metode strip() dari string untuk menghapus karakter baris baru di akhir data, dan selanjutnya menggunakan metode split() untuk memisahkan dengan koma.

7.4 Pengolahan data dua dimensi
7.4.1 Representasi data dua dimensi
Data dua dimensi terdiri dari beberapa data satu dimensi dan dapat dianggap sebagai kombinasi data satu dimensi. Oleh karena itu, data dua dimensi dapat direpresentasikan dengan daftar dua dimensi, yaitu setiap elemen daftar berhubungan dengan deretan data dua dimensi. Elemen ini sendiri juga merupakan tipe daftar, dan setiap elemen di dalamnya sesuai dengan setiap nilai kolom di baris ini.
Tip: Tipe data untuk data dua dimensi
Data dua dimensi umumnya menggunakan tipe data yang sama untuk menyimpan data agar mudah dioperasikan.
Meskipun terdapat banyak sekali nilai numerik pada data dua dimensi, karena baris atau kolom nilai numerik pada contoh ini tidak semuanya berupa angka, melainkan teks karakter, maka nilai numerik tersebut dinyatakan secara seragam sebagai string.
